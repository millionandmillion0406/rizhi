## 🔴 五条铁规（工程纪律）

1. **改前先读** — 动手改服务器代码前，先 `cat` 全文读完
2. **改后验语法** — 每次修改后跑 `python -m py_compile`
3. **部署验存活** — 部署后 `curl` 确认 HTTP 200
4. **一次一处** — 一次只改一个地方，验完再改下一处
5. **每次留记录** — 改了什么、怎么改的、结果写进日志

**核心思维：一口气改三处一起部署 = 一起炸 = 不知道哪错了。**

## ✅ 今后工作准则

1. **先搜再做** — 做项目前先上 GitHub 搜类似开源项目，借鉴学习再行动
2. **先 Spec 后代码** — 动手前先写清晰需求，用户确认后再开工（来自 Superpowers ⭐250K）
3. **少即是多（Ponytail 七级阶梯）** — 写代码前爬梯子：YAGNI → 复用 → 标准库 → 原生 → 已有依赖 → 一行 → 最小实现
4. **修根因不修症状** — Bug 修复先 grep 所有调用者，一次性修共享函数
5. **反 AI 味** — 输出要有品味，不模板化、不堆砌（来自 Taste Skill ⭐61K）
6. **记忆同步** — 本文件 ↔ 本地 AGENTS_MEMORY.md 保持同步
7. 每次操作前想"这个该记到哪里"
8. 大操作先讨论方案再动手
9. 代码推 GitHub 前互相审查
10. 实用第一，高级感/极简审美第二
11. 凭证用环境变量，绝不写死在代码里
12. 非平凡逻辑留一个可运行的检查（assert / 小测试）
