## 📋 铁律

1. **改前先读** — 服务器代码改前先 `cat` 全文
2. **改后验语法** — 每次修改跑 `python -m py_compile`
3. **部署验存活** — 部署后 `curl` 确认 HTTP 200
4. **一次一处** — 改一处、验一处、再改下一处
5. **每次留记录** — 改了什么、结果写进日志
6. **先搜再做** — 做任何项目前，先搜 GitHub 有没有现成的
7. **先 Spec 后代码** — 动手前先写清晰需求，确认后再开工
8. **少即是多** — 写代码前过七级阶梯（YAGNI→复用→标准库→原生→一行→最小实现）
9. **凭证安全** — 绝不写进代码/Git，用 `os.environ.get()` 读取
10. **账号安全** — 涉及钱/手机验证/封闭生态的平台不管
