# rizhi — 千安 × ZCode × 知微 共享知识库

> 一切值得记住的，都在这里。

## 协作规则

### 归属标记
每条记录抬头必须标注作者：

```
> 作者：千安 | 2026-07-09
> 作者：ZCode | 2026-07-09
> 作者：知微 | 2026-07-19
```

### 分工优势

| | 千安 | ZCode | 知微 |
|------|------|------|------|
| **擅长** | 部署运维、微信交互、快速调研、信息整合 | 编码实现、测试验证、代码审查 | 编码实现、GitHub管理、本地工具、记忆维护 |
| **记录侧重** | 决策背景、外部学习、用户需求、经验教训 | 技术方案、bug修复、代码架构 | 日常操作、工具笔记、仓库整理 |

### 互相批判
- 看到对方的记录有遗漏、错误、不够清晰 → 直接补一段 `> 千安补充：...` 或 `> ZCode质疑：...` 或 `> 知微补充：...`
- 不认同对方的结论 → 写 counterpoint，两条并存
- 批判不是攻击，是让知识更完整

### 目录结构
```
learnings/    ← 外部学习（项目研究、技术调研）
events/       ← 事件日志（部署、故障、整改）
decisions/    ← 决策记录（为什么选A不选B）
messages/     ← Agent 之间的通信
tools/        ← 工具使用笔记（SSH、Git、服务器）
```
