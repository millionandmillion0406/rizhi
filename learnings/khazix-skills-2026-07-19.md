## 🎯 5 个 Skill 详解

### 1. 💽 storage-analyzer（清理垃圾）
**一句话**：扫描 Mac/Windows 整机磁盘，三色分级给清理决策，网页上一键移废纸篓
**核心设计**：
- 🟢 **绿灯** — 纯缓存，可一键清
- 🟡 **黄灯** — 含用户数据，只给"打开/移废纸篓"
- 🔴 **红灯** — 系统文件，只能打开看看
- **铁律**：全程只读，删除必须用户在浏览器点按钮+二次确认
- 跨平台（macOS 已实测，Windows 代码就绪）

### 2. 🔥 aihot（AI HOT 资讯查询）
**一句话**：让 Agent 用一句话拿到 AI 日报和动态，无需 API Key
**核心设计**：
- 通过 `aihot.virxact.com` 公开只读 API 获取数据
- 不凭训练记忆回答新闻
- 严格的 User-Agent 要求 + 版本自检机制
- 按分类拉：模型/产品/行业/论文/技巧

### 3. 🧹 neat-freak（洁癖）
**一句话**：干完活跑 `/neat`，自动对齐项目文档、CLAUDE.md、Agent 记忆，审计规则执行
**核心设计**：
- 三层对齐：CLAUDE.md（给AI看）、docs/README（给人看）、Agent 记忆（跨会话）
- 规则审计：检查 CLAUDE.md/AGENTS.md 是否同源、必备文件有无缺失、引用路径是否还在
- 触发词极广，几乎涵盖所有"收尾"场景

### 4. 🔭 hv-analysis（横纵分析法）
**一句话**：丢给它一个产品/公司/概念，给你一份万字 PDF 研究报告
**核心设计**：
- **纵轴**：从诞生讲到当下，像讲故事
- **横轴**：同期竞品逐一对比
- 最终交叉产出独到洞察
- 产出：10,000~30,000 字精美 PDF 报告

### 5. ✍️ khazix-writer（卡兹克写作）
**一句话**：装上后 Agent 用卡兹克的口吻和节奏写公众号长文
**核心设计**：
- 完整的写作风格规则（节奏、叙事、判断、修辞）
- 四层自检体系（结构、节奏、内容、文字）
- 会**拒绝**写「赋能、抓手、闭环」等词

## 💡 关键学到的东西

### 1. SKILL.md 的设计标准
- 每个 SKILL.md 都有 **frontmatter**（name + description + 触发词）
- description 里写明所有触发场景，避免漏触发
- 正文分：为什么重要、执行流程、安全边界、输出格式

### 2. 安全模型设计
- storage-analyzer 的**三色分级 + 二次确认**是优秀的安全模式
- 所有写操作走"展示不执行"原则
- 全程只读，需要用户主动确认才放行

### 3. 跨平台兼容
- 每个 skill 都标注兼容哪些 Agent 平台
- 用标准 Markdown 格式，不绑定特定 Agent

### 4. 触发词设计
- 覆盖中英文、口语化表达
- 连口语误用（如"内存"指"存储"）都考虑到了

### 5. neat-freak 的审计机制
- 规则不落地 = 装饰品
- 三层受众不同、职责不重叠
- 审计比对齐更重要：检查规则有没有被真正执行

## 📌 对我们项目的启发

1. **我们可以把常用操作也写成 Skill** — 比如 JM PDF 下载、知识库查询
2. **安全三色分级** — 适合用到我们的工具代码中
3. **neat-freak 的理念** — 每次会话结束做文档对齐，可以集成到我们的工作流
4. **khazix-writer 的风格规则** — 我们写日志时也可以学这种结构化风格定义
5. **触发词覆盖广** — 让技能更容易被自然语言触发
