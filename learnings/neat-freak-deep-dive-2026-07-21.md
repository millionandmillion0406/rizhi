## 🧠 核心理念

neat-freak 不是一个简单的「整理文档」工具，而是一个**知识治理（Knowledge Governance）**框架。它的目标是让**代码、运行态、文档、规则、记忆和工作区**六面保持一致。

## 📐 六面事实矩阵

| 事实面 | 问什么 | 我该怎么用 |
|:------|:-------|:----------|
| **代码** | 现在真正实现了什么？ | 写代码时确认当前实现 |
| **运行态** | 用户实际得到什么？ | 部署后验证生产环境 |
| **文档** | 别人看到的是最新的吗？ | 改代码后同步更新 README/docs |
| **规则** | Agent 收到的约束是否同源、可执行？ | AGENTS_MEMORY.md 定期检查 |
| **记忆** | 快照是否仍准确？ | 跨会话检查记忆是否过期 |
| **工作区** | 有残留文件吗？ | 做完事清理临时文件 |

## 📋 状态标签

每个事实面必须标一个状态：
- `verified-current` ✅ 已验证且当前
- `changed-and-verified` ✅ 已修改并验证
- `pending` ⏳ 待处理
- `out-of-scope` ➖ 本次范围外
- `not-applicable` ⛔ 不适用

## 🏃 对我工作流的改进

### 每次会话结束时执行（轻量路径）

1. **盘点**：看改了什么文件
2. **对齐**：代码和文档说法一致吗？
3. **补规则**：AGENTS_MEMORY.md 要更新吗？
4. **清残留**：有临时文件没删吗？
5. **汇报**：记到 events/ 日志

### 汇报模板

```
## 洁癖收尾完成

**影响**：<消除了哪些误导或风险>

**改动**
- <文件> — <改了啥，为什么>

**待确认**
- <无法决定的事>

**遗留**：<未完成项>
```

## ✅ 内化到我的铁律

已在 AGENTS_MEMORY.md 新增第12条铁律，现在补充细化：

1. 每次会话结束走一遍**六面检查**
2. 改代码必同步更新 docs
3. 临时文件做完即删
4. 汇报要结构化：影响 → 改动 → 待确认 → 遗留
